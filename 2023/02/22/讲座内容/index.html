语义通信技术的发展趋势和应对之策-2023.2.8–张平

研究依据和研究意义

  1. 现代信息论：理论、数学角度–传统的通信大厦。算法算力支撑下，进行语义通信的研究，从无源到有源进行发展。

  2. 研究的意义：
    （1）传统通信解决信息搬迁的问题。信源如何进行度量？信道的传输能力？–以熵为基础.
    —-带来的问题：通信变成了工程问题–华为制造业等只是实现了这些工程。
    —-面临的挑战：增加信息传输的物理维度、改变资源挖掘利用的方式来提升系统的通信容量—-天线数目过多、优质频谱稀缺（频段高采样率增加、损耗增加）、通信能耗增加。采用并行的方式进行堆叠式的创新不断逼近传输的极限。
    （2）现代通信，新兴的通信对象是机器，人是有意识的能够理解一些内容，但是机器理解符号是有所差异的。机器之间的类人通信需要交互意图，而不是传输字符。否则机器需要进行很多的语言理解，这个成本是很高的。
    （3）新型通信需要进行颠覆性创新，建立新理论、新型的技术体系，形成未来通信发展途径。
    （4）语义信息：信息的含义传输。考虑信息的含义，关注通信含义的准确传播。—-注意力集中在需要关注的部分，关注通信含义的准确传递。语义信息包含人的情感会非常复杂，使用语法传输只是进行压缩的本质。
    （5）语用信息：信息效用的传输。
    （6）语义通信的历史研究：Weaver、北邮魏玲等等。假定存在一些知识库，发一个简单的字符即可传递一个复杂的信息。

  3. 研究内容：设计思想、基础理论、核心方法–实现信息的简约表征和高效的传输
    （1）信源–语义信息的提取和表征（发端语义知识库）–语义信息编码（发端语义知识库）—-（语法通信的信源信道编码–channel–信源信道译码）–语义信息解码（收端语义知识库）–语义信息的综合和重建（收端语义知识库）–信宿
    通信人需要关联信道，确定传输的效果–计算机不考虑信道，只做语义的内容
    (2)设计思想：以语义信息为对象，以语义知识库为关键支撑，重塑通信系统的信息提取、表征、编译码、综合与重建等核心环节
    (3)基础理论：解决“语义信息度量”、“语义传输容量”通信基础理论伺题
    (4)核心方法：面向语义任务、信道能力的弹性信息压缩，语义信源-信道-信宿联合设计（信源–汉明距离，调制–高斯距离–如何统一，光通信–高斯信道汉明距和高斯距一样）：可能不需要AD、DA的方式，通过IO传递直接调制到信道上
    （5）构建数据、模型协同驱动的方式：挖掘数据含义，以数据打破模型局限–现在的模型基于统计规律，未考虑数据的潜在规律；模型驱动是现在使用的，需要克服数据困境，使用模型补偿数据的不足。
    （6）极端复杂的环境下，5G的技术不能维持通信，信号的大幅衰落引发中断效果，复杂环境下需要以平均模型为主，特殊环境下使用数据要素弥补模型局限，实现粗细粒度的均衡
    （7）经典信息论：熵、率失真、信道容量、互信息–>上升到语义。

  4. 6G

    （2）垂直行业赋能：工业互联网、智能无人系统–单一的场景

  3. 语义通信：面向任务需求的通信。收端明确任务之后，只需要识别图片中的内容，传输时就不需要传输完整的图片，只传输与任务相关的信息。–最重要的区别就是引入了语义知识库，在语义知识库支持下，对传输的信息进行表征。
  4. 语义知识库是什么？
  5. 语义通信：数据高质量恢复、面向需求的恢复
  6. 同声传译功能–任务需求：选择具体的频道–具体的任务（用户需求多样化），或者多角度接入会议（场景是多样化的），此时需要传输的数据是巨大的，现有的网络无法承载如此巨大的数据–探索新的方法，用于减少需要传输的数据量，但是能保障语义信息的交互
  7. 理论研究：（1）信息论的角度、统计理论的研究–可以提出各种框架，但是认可度是存疑的。（2）信源内部的语义特征–语义熵的编码：图片领域中存在结构化编码。（3）传输层面：信源编码前面添加语义信息提取，但是这样传输端没有进行很好的适配，传输有效性没有达到最优，因此传输需要考虑信道（4）语义特征的测量:语义噪声如何定义（5）方法：深度学习驱动的方法进行语义信息提取，但是目前理论不完善

文字传输 DEEPSC–text transmission

  1. 发端：语义编码器、信道编码器（基于背景知识库）
  2. 收端：信道解码器、语义解码器（基于背景知识库）
  3. 收发目的：语义信息传输后基本保持一致。
  4. 两种类型的信道–物理信道（物理传输需要考虑的噪声）、虚拟信道（语义信道–存在语义噪声：收发端对语义信息理解的不同导致的噪声）
  5. 语义编码：采用transformer架构–扩展到了多个模态。语义编码器理解话语的含义
  6. 联合的S、C编码–（1）误差函数：语义误差cross entropy+通信系统的传输速率（channel encoder性能差于联合编码的设计–考虑语义信息的重要性，提供不等差错的保护）（2）性能度量：bleu score（文字层面） + sentence similarity（基于bert模型映射到语义向量空间）
  7. deep sc考虑动态环境–（1）动态变化的背景知识–语义知识库–动态变化任务–更新 source encoder（2）信道环境变化–更新channel encoder，以上两种情况均采用transfer learning的方式
  8. 低信噪比有很好的鲁棒性，动态环境变化也有适应性

semantic communication 用于sppech

  1. deepsc-st：语音信号处理。传输端采用的策略基于任务驱动型。
  2. 场景：语音输出，语音转化文本。不同的任务场景选择不同的特征进行传输，减少传输需要的数据量–选择不同的语义编码模块进行传输。具体来说，用户输入之后提取特征再生成相似的信号，传输时可以减少传输的数据量
  3. deepsc-st通信需要传输的数据量，可以减少90%左右

semantic communication 用于多模态数据

  1. Mu-DEEPSC：视频流的传输、多模态数据的传输–多用户的场景
  2. single-model multi-user：用户对应一种应用，设计的模块需要独立开来
  3. multi-model multi-user：收端执行VQA任务–发送文字、图片基于图片提出问题并进行回答。收端不恢复文字、图片，直接给出回答即可。编码时需要知道对方需要满足的任务，将相关的特征进行提取，进一步降低需要传输的数据量（jpeg+ldpc–图片，文字：utf-8+turbo码）
  4. 效果：低信噪比下效果好，同时数据量少

    发送端在不知道发送任务的情况下保留了语义信息。

  5. 数据自适应网络–观测数据、训练数据不一样，使用迁移学习的域自适应的方法，迁移观测数据到训练的数据库，只需要发端单独执行，接收端不需要参与执行，训练好的语义编译码器不需要重新训练。

    基于循环对称神经网络来做，观测数据经过生成器转移到经验数据库，再进一步到另外的生成器转移到观测空间，经过一个辨别器区分原始数据和经过转换后的数据，直到辨别器无法区分。
    训练过程不需要太多的数据，不需要更新原先的语义编码网络。只在发送端更新，保证了收发双方的隐私。

语法通信的演进和局限

  1. 水、电、信息成为生活的基本需求。5G-AR、VR、车联网等，人与信息融为一体
  2. 信息形态：生物信息、文字、图片、声音、视频。信息终端：电脑、收集、平板、车载
  3. 信息熵的来源

    信源符号压缩为新的编码符号，每个新原符号需要的编码符号平均长度的最小值为信源的信息熵
  4. 信息熵推广到信息论（互信息）–工程指导

  5. 香农信息熵基于概率度量语法信息，但没有语义信息的提取，可以实现信息的继续压缩

语义通信的思想和范畴

  1. 语义信息传输：信息含义传输；语用信息传输：信息效用传输
  2. 语法通信发展逼近极限

    压缩逼近压缩极限，传输通过增加维度来逼近极限–难以持续发展
  3. 香农1948：不意味着人们要永远忽略信息的含义，通信的本源需要知道传输信息的含义。语义信息是什么？
  4. 语义信息是一种简洁的表达–减少数据量的传输
  5. 语义通信的架构模型

    （1）收发端知识库：双方的背景知识–传输少量信息理解高层的含义
    （2）需要信源信道一体化设计：语义提取的表征需要关注恶劣情况、无线情况下的通信传输问题–如果一个核心参数丢失，那整个的语义内容能否被恢复

语义通信进展和趋势

  1. 语义通信研究内容
    （1）语义通信如何被度量

    主观色彩和知识库是关联的–同一事物的不同理解，信息存在差异，该如何度量？是否有客观的部分并进行度量？
    （2）语义信息如何压缩–接受任务语义的需求是相关的

    （3）语义信息如何压缩—跨模态传输

    （4）语义信息如和传输

    语法通信关注模块化的设计，语义传输让信源的统计特性发生变化，再做模块化的设计能否达到最优？–一体化设计：语义提取、表征编码、信道编码进行一体化设计（可能会变换新的思路：基于统计特性、统计信号处理、经典信号处理转变到人工智能、神经网络、深度学习、NLP的演进，可以做到一体化的设计是可能的，效率是更高的）
    （5）语义信息传输的模型扩展–语义信息的度量方式：BER到语义相似度或失真度作为判断准则

  3. 语义通信未来的潜在应用

研讨

  1. Q：如何判断图像、视频语义压缩的细节？给予不同的编码权重？A：简单的方式：语义分割，在挑选出重点的语义，但是这样是人工干预的、粗糙的；可以利用神经网络资助的学习出来特征的重要度、再联合信道编码进行设计。AA：语义通信不是语义分割，语义通信需要信道传输特征，各个特征提取需要考虑信道响应特征来分配权重（学习信源、信道特征–更多是做一体化的设计进行传输）
  2. Q：（方敏）一体化设计：语义噪声、语义编码对传统信源信道编码的影响是什么？A：分两个层面来讲，（1）传统的模块化设计对传统系统影响不会太大，输入信源当成bit来做（2）一体化设计是一个主流，语义噪声不会有严格界限，语义提取、编码、信道编码不严格区分。语义噪声–语义提取表征产生的失真的地方。AA：在现有基础上，如何嵌入新型的网络–面向机器、面向人的网络–可能存在多种场景融合、特殊场景部署新的网络效率更高
  3. Q：语义通信的容错能力怎么理解？A：语义本身理解为上下文的关系或者含义或者背景提供的先验知识，允许接收端错误接受也能恢复。可能少了一些东西，基于背景知识也能进行差错恢复。
  4. Q：在无线衰落信道下，信源信道联合编码能克服信道的影响吗？能和ldpc融合吗？A：信道衰落到什么尺度，在联合也达到不到需求，但是不是很严重，可以对差错进行抵抗。AA：语义通信和传统通信是可以融合的，另外LDPC的功能可以用神经网络进行表示，进一步一体化进行优化。
  5. Q：不同带宽的条件下，尤其是较小带宽，语义通信对信源关键信息的压怎么处理？A：语义压缩编码需要考虑信道带宽，基于信道考虑分配多少比特的信息给到细节的内容，带宽充足突出细节，带宽不足只传输部分细节。因此语义通信可以适配恶劣的无线信道，用于对抗悬崖效应。–泛化对不同信道的适配能力
